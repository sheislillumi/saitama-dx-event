# 県内企業DX推進統合イベント運営システム 開発仕様書
## (Claude Codeへの指示用ドキュメント)

対象イベント: 県内企業DX推進に係る統合イベント(2027年1月20日・ソニックシティ)
作成目的: イベントHP／出展申込サイト／来場者申込サイト／QRコード受付・管理ダッシュボードの構築

---

## 1. システム概要

出展者・来場者からの申込を受け付け、来場者には確認メールでQRコードを送付、当日はQRコードを読み取って受付(チェックイン)を行い、その状況を管理ダッシュボードでリアルタイムに確認できるシステムを構築する。

### 主要4画面
1. イベント公式HP(概要・アクセス・各申込への導線)
2. 出展申込フォーム
3. 来場者申込フォーム(登録完了時にQRコード付き確認メールを送信)
4. QRコード受付画面(当日、会場スタッフが使用)
5. 管理ダッシュボード(出展者・来場者一覧、チェックイン集計、CSV出力)

## 2. 技術構成

- フロントエンド: Vanilla HTML/CSS/JS(フレームワーク不使用)。GitHub Pagesでホスティング想定
- バックエンド: Google Apps Script(GAS)をWeb Appとしてデプロイし、API窓口とする
- データベース: Googleスプレッドシート(出展者シート・来場者シート)
- メール送信: GAS標準のGmailApp
- QRコード生成: Google Chart API または qrcode.js(クライアント側生成でも可)
- QRコード読み取り: html5-qrcode(JSライブラリ、カメラアクセス)
- ホスティング: GitHub Pages(フロント)+ GAS Web App(バックエンド)

この構成を選んだ理由:
- 追加のサーバー費用が発生しない
- 既存のGoogleスプレッドシート運用(来場者チェックイン・差分CSV抽出)とそのまま連携できる
- 開発規模に対して学習コストが低い

将来的に来場者数が数千人規模になる、あるいは同時アクセスが多く発生する場合は、Firebase(Firestore + Cloud Functions + Hosting)への移行を検討する。

## 5. 各画面の要件

### 5-1. イベントHP(index.html)
- イベント概要、日時、会場、アクセス
- 展示ゾーン紹介、DX大賞表彰式の案内
- 出展申込・来場者申込への導線(ボタンリンク)
- スマホ・PC両対応のレスポンシブデザイン

### 5-2. 出展申込フォーム(exhibitor/index.html)
- 入力項目は「4. データスキーマ」のExhibitorsシートに準拠
- 送信時: GAS APIへPOST → スプレッドシートに1行追加 → 自動返信メール(「申請を受け付けました」)
- 必須項目チェック・メール形式チェックなどのバリデーション

### 5-3. 来場者申込フォーム(visitor/index.html)
- 入力項目は「4. データスキーマ」のVisitorsシートに準拠
- 送信時: GAS APIへPOST → UUID・QRトークンを発行 → スプレッドシートに登録 → QRコード画像を生成し確認メールに添付して送信
- 個人情報の取り扱いに関する同意チェックボックスを必須にする

### 5-4. QRコード受付画面(checkin/index.html)
- html5-qrcodeでカメラを起動し、QRコードを読み取る
- 読み取り成功時: トークンをGAS APIへPOST → 該当来場者を検索 → チェックイン状態を更新 →「〇〇様 受付完了」を画面表示
- 重複スキャン時は「受付済みです」と表示
- 未登録トークンの場合はエラー表示+手動登録への導線
- 通信不安定時のフォールバックとして、トークンやID手入力にも対応する

### 5-5. 管理ダッシュボード(admin/index.html)
- 簡易パスワード認証(GASのPropertiesServiceで管理)
- 来場者一覧(検索・フィルタ・ソート)
- リアルタイム集計(申込者数、チェックイン済み人数、チェックイン率)
- 出展者一覧・ステータス管理
- CSV出力機能(既存の差分抽出運用に合わせ、出力日時を記録できるようにする)

## 6. GAS側API仕様(gas/Code.gs)

- `doPost(e)`: パラメータ`action`で分岐
  - `registerExhibitor` — 出展申込の登録
  - `registerVisitor` — 来場者申込の登録、QRトークン発行、確認メール送信
  - `checkin` — QRトークンを受け取り、該当来場者のチェックイン状態を更新
- `doGet(e)`: パラメータ`action`で分岐
  - `getVisitors` — 管理画面用(要認証)
  - `getExhibitors` — 管理画面用(要認証)
  - `getStats` — 集計データ取得
- 管理系API(`getVisitors`等)には簡易APIキーやパスワードをパラメータまたはヘッダーで検証する処理を入れる

## 7. セキュリティ・個人情報保護の考慮事項

- QRコードには個人情報を含めず、ランダムなUUID/トークンのみを埋め込む
- 管理ダッシュボードは必ずパスワード保護し、誰でもアクセスできる状態にしない
- スプレッドシートの共有設定は関係者のみに限定する
- 通信は必ずHTTPS
- フォーム上に個人情報の利用目的を明記し、同意を得る
- イベント終了後のデータ保管・破棄ルールをあらかじめ決めておく

## 9. 補足

- 出展者の募集・選定自体は委託者(公益財団法人埼玉県産業振興公社)が行う業務のため、
  出展申込フォームは「申請の受付」までを担う位置づけとする(選定結果の反映はステータス列で管理)。
- 来場者アンケートは「単純集計までを受託者が行う」仕様のため、管理ダッシュボードには
  詳細な分析機能までは含めず、集計結果のCSV出力までを実装範囲とする。
- 展示ゾーン受付と講演会場受付を分けて管理する場合は、Visitorsシートの「受付場所」列を
  チェックイン時に判定・記録する処理を追加する(QRコード受付画面側で受付場所を選択する
  UIを用意するなど)。
